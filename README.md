# 🕌 بوت تيليجرام - آيات وأحاديث وأدعية (توجه شيعي)

بوت بايثون (3.11.9) ينشر تلقائيًا آيات قرآنية، أحاديث عن رسول الله ﷺ وأهل البيت
عليهم السلام، أدعية، وأذكار (باقيات صالحات) - في أي قناة تيليجرام يُضاف إليها
كأدمن، بالتزامن مع أوقات الصلاة الخمسة (محسوبة بمنهج الشيعة الاثنا عشرية عبر
Aladhan API المجاني).

**شرط الاستخدام:** أي شخص يريد تفعيل البوت في قناته، يجب أن يكون مشتركًا في
القناة الرسمية أولًا - وهذا الشرط يُفحص بشكل حي ومستمر، وليس مرة واحدة فقط.

## 🧩 كيف يعمل البوت (باختصار)

1. أي مستخدم يضيف البوت كأدمن (بصلاحية إرسال الرسائل) في قناته.
2. يتحقق البوت فورًا: هل هذا المستخدم مشترك في القناة الرسمية؟
   - **نعم** → يُفعَّل النشر في قناته تلقائيًا.
   - **لا** → لا يُفعَّل، ويُرسَل له تنبيه في الخاص لطلب الاشتراك أولًا.
3. عند كل وقت من أوقات الصلاة الخمسة، **قبل** كل عملية نشر، يعيد البوت
   التحقق من اشتراك كل مالك قناة. إن غادر القناة الرسمية، يتوقف النشر في
   قناته فورًا حتى يعود ويرسل `/reactivate`.

## ⚙️ خطوات الإعداد الكامل (خطوة بخطوة)

### 1) إنشاء البوت عبر BotFather

1. افتح محادثة مع [@BotFather](https://t.me/BotFather) في تيليجرام.
2. أرسل `/newbot` واتبع التعليمات (اسم البوت، ثم username ينتهي بـ `bot`).
3. سيعطيك BotFather **توكنًا** (Token) بالشكل: `123456789:AAExample...`
   - **احتفظ به سريًا تمامًا ولا تشاركه مع أحد أو ترفعه على GitHub.**

### 2) الحصول على معرفك الرقمي (ADMIN_ID)

1. افتح محادثة مع [@userinfobot](https://t.me/userinfobot).
2. أرسل أي رسالة، سيرد عليك برقم معرفك (ID) - هذا هو `ADMIN_ID`.

### 3) إعداد القناة الرسمية (forca91)

1. أضف البوت الذي أنشأته للتو **كأدمن** في قناة `t.me/forca91` نفسها،
   مع صلاحية "إضافة أعضاء" أو أي صلاحية إدارية أساسية (لا يحتاج صلاحية النشر هنا،
   فقط يحتاج أن يكون أدمن ليستطيع التحقق من قائمة الأعضاء).
   - ⚠️ **هذه الخطوة إجبارية** - بدونها لن تعمل آلية الاشتراك الإجباري إطلاقًا،
     والبوت سيرسل لك تحذيرًا تلقائيًا عند التشغيل إن نسيتها.

### 4) تجهيز ملف الإعدادات محليًا (للتجربة قبل الرفع)

```bash
cp .env.example .env
```
افتح `.env` واملأ:
```
BOT_TOKEN=التوكن_من_BotFather
ADMIN_ID=معرفك_الرقمي
OFFICIAL_CHANNEL_USERNAME=forca91
STATE_DIR=./state
```

### 5) التشغيل المحلي (اختياري - للتجربة قبل الاستضافة)

```bash
python3.11 -m venv venv
source venv/bin/activate          # على ويندوز: venv\Scripts\activate
pip install -r requirements.txt
python -m bot.main
```

### 6) تشغيل الاختبارات (للتأكد أن كل شيء سليم)

```bash
pip install pytest pytest-asyncio
python -m pytest tests/ -v
```
يجب أن تشاهد `14 passed`.

## 🚀 الرفع على GitHub ثم Railway

### أ) رفع المشروع على GitHub

```bash
cd telegram-quran-bot
git init
git add .
git commit -m "النسخة الأولى من بوت الآيات والأحاديث"
git branch -M main
git remote add origin https://github.com/USERNAME/telegram-quran-bot.git
git push -u origin main
```
> ملف `.gitignore` جاهز مسبقًا ويستثني `.env` و `state/` تلقائيًا، فلن يُرفع أي سر بالخطأ.

### ب) الربط مع Railway

1. ادخل إلى [railway.app](https://railway.app) وسجّل الدخول (يفضّل عبر GitHub).
2. اضغط **New Project** → **Deploy from GitHub repo** → اختر المستودع الذي رفعته.
3. Railway سيكتشف تلقائيًا أنه مشروع بايثون (بفضل `railway.json` و `runtime.txt` الجاهزين).

### ج) إضافة متغيرات البيئة في Railway

من صفحة المشروع → تبويب **Variables** → أضف كل هذه المتغيرات (بنفس الأسماء):

| المتغير | القيمة |
|---|---|
| `BOT_TOKEN` | التوكن من BotFather |
| `ADMIN_ID` | معرفك الرقمي |
| `OFFICIAL_CHANNEL_USERNAME` | `forca91` |
| `STATE_DIR` | `/data` (انظر الخطوة التالية) |
| `PRAYER_LATITUDE` | `33.3152` (اتركه فارغًا لاستخدام الافتراضي - مركز بغداد بدقة عالية) |
| `PRAYER_LONGITUDE` | `44.3661` (اتركه فارغًا لاستخدام الافتراضي) |

### د) ⚠️ خطوة حرجة: إنشاء Volume دائم

بدون هذه الخطوة، ستُفقد قائمة القنوات المسجَّلة وتتبع المحتوى المنشور
في كل مرة يُعاد فيها نشر المشروع (redeploy) على Railway:

1. من صفحة الخدمة (Service) في Railway → تبويب **Settings** → قسم **Volumes**.
2. اضغط **New Volume**.
3. اجعل مسار التركيب (Mount Path) = `/data` (نفس قيمة `STATE_DIR` أعلاه بالضبط).
4. احفظ، وسيُعاد نشر الخدمة تلقائيًا.

### هـ) التشغيل

بعد ضبط المتغيرات وإنشاء الـ Volume، اضغط **Deploy**. راقب الـ **Logs**:
- إذا رأيت `تم التحقق: البوت أدمن في القناة الرسمية ✅` فكل شيء سليم.
- إذا رأيت تحذيرًا بأن البوت ليس أدمن، ارجع للخطوة (3) وتأكدها.

## 🧪 كيفية الاختبار الفعلي بعد النشر

1. أرسل `/start` للبوت في الخاص للتأكد أنه يستجيب.
2. بصفتك الأدمن (`ADMIN_ID`)، أرسل `/test` لمعاينة شكل الرسالة في الخاص
   قبل أي نشر فعلي.
3. أنشئ قناة تجريبية خاصة بك، أضف البوت كأدمن فيها بصلاحية النشر.
   - إن كنت مشتركًا في `forca91`، سيصلك تأكيد فوري بتفعيل قناتك.
4. كأدمن، أرسل `/postnow` لتجربة نشر فوري في كل القنوات النشطة دون
   انتظار وقت الصلاة القادم.
5. أرسل `/status` لرؤية عدد القنوات النشطة والمتوقفة.

## ➕ كيفية إضافة المزيد من الآيات/الأحاديث/الأدعية لاحقًا

كل الملفات في `data/` هي JSON بسيط. لإضافة عنصر جديد، انسخ نفس الشكل
وأضف `id` فريدًا لم يُستخدم من قبل، مثال لإضافة حديث جديد في `ahadith.json`:

```json
{"id": "h031", "text": "نص الحديث هنا", "narrator": "الإمام الصادق عليه السلام", "source": "الكافي", "category": "الصبر"}
```
لا حاجة لأي تعديل في الكود - البوت سيلتقطه تلقائيًا عند إعادة التشغيل.

## 🔒 ملاحظات أمنية مهمة

- لا تشارك `BOT_TOKEN` مع أي أحد؛ من يملكه يتحكم بالبوت بالكامل.
- إذا شككت أن التوكن تسرّب، اذهب لـ BotFather وأرسل `/revoke` لتوليد توكن جديد فورًا.
- كل الأسرار تُقرأ من متغيرات البيئة فقط - لا يوجد أي سر مكتوب داخل الكود.
- ملف `.gitignore` يمنع رفع `.env` و `state/` بالخطأ إلى GitHub.
